# Biggest remaining issue:

# It still feels TABLE-based instead of LEDGER-based

This is the key difference.

Right now:

```text
Service Lines Table
Posting Ledger Table
CAS Table
```

Epic-style systems feel like:

> one continuous financial transaction stream.

---

# What to change next

# 1. Merge Service Lines + Posting Ledger conceptually

Right now they feel disconnected.

Instead:

```text
▶ DOS 05/19/2026 • CPT 90837
--------------------------------
Charge                  +175.00
Allowed Amount          140.00
Insurance Payment      -140.00
Contractual Adj         -35.00
Patient Resp             35.00
--------------------------------
Balance                   0.00
```

THAT feels like Epic.

Financially self-contained rows.

---

# 2. Introduce expandable ledger trees

This is the single biggest UX upgrade left.

Current:

* flat rows

Needed:

* hierarchical financial drilldown

Example:

```text
▶ Claim CLM-2026-004
```

Expand:

```text
▶ DOS 05/19 CPT 90837
    Charge                  +175
    Insurance Payment       -140
    CO-45 Adjustment         -35
    Patient Responsibility   +35
```

This creates:

* traceability
* confidence
* enterprise feel

---

# 3. Financial signs/colors are missing

Huge issue psychologically.

Everything visually looks neutral.

Enterprise ledgers use:

* positives
* negatives
* debits
* credits
* balances

visually.

---

## Example

```text
Charge                +175.00
Insurance Payment     -140.00
Adjustment             -35.00
Patient Resp           +35.00
```

with:

* green credits
* muted adjustments
* orange patient balances
* red unresolved balances

This dramatically changes the feel.

---

# 4. Missing balancing rail

You NEED a sticky posting summary.

Right now the user must mentally calculate status.

Epic never does that.

---

## Add sticky sidebar or footer

```text
ERA Total           $168.00
Applied             $168.00
Remaining             $0.00

✓ Balanced
```

OR:

```text
$42 remaining to apply
```

This is extremely important psychologically.

---

# 6. Need inline posting interactions

Right now it appears mostly read-only.

Enterprise systems feel:

* editable
* live
* interactive

---

## Needed interactions

Inline:

* apply adjustment
* split payment
* transfer balance
* edit patient resp
* resolve unmatched

without modals.

---

# 7. CAS adjustments should integrate INTO the ledger

Right now CAS section feels detached.

Epic-style systems integrate CARC/CAS directly into transactions.

Example:

```text
Insurance Payment       -140
CO-45 Adj                -35
```

Not:

* separate table below

This matters a LOT.

---

# 8. Need transactional chronology

This is one of the LAST major pieces.

Enterprise ledgers are chronological.

You need:

```text
05/01 Charge posted
05/12 Claim submitted
05/18 ERA received
05/18 Payment applied
05/18 Adjustment posted
05/18 Patient balance created
```

This creates:

* auditability
* financial trust
* enterprise feel

---

# 9. Typography hierarchy still needs work

Money still doesn’t dominate enough.

Amounts should visually drive the screen.

Increase:

* amount size
* weight
* contrast

Reduce:

* labels
* metadata emphasis

---

# 10. The UI still feels “web app clean”

This is subtle but important.

Epic/PracticeSuite intentionally use:

* tighter spacing
* denser rows
* reduced padding
* compressed interaction zones

because billers prioritize:

* speed
* throughput
* scanning

You should move about:

* 15–20% denser

especially:

* row height
* panel spacing
* table padding
