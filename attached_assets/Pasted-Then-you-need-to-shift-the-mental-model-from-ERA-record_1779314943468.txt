# UI Changes You Need

# 1. Make the ledger the center of the screen

Right now:

* summary dominates
* ledger secondary

Reverse that.

The ledger IS the product.

---

## Epic-like layout

```text
------------------------------------------------
ERA HEADER
Payer | Check # | Amount | Status
------------------------------------------------

LEDGER GRID
------------------------------------------------
DOS | CPT | Charge | Allowed | Ins Paid | Adj | Pt Resp | Balance
------------------------------------------------

TRANSACTION DETAIL PANEL
------------------------------------------------
Timeline / Notes / Audit / Actions
```

The grid becomes dominant.

---

# 2. Introduce true transactional rows

Instead of:

```text
Claim row
```

Use:

```text
transaction row
```

Example:

```text
05/12  CHARGE        90837     +$180
05/19  INS PAYMENT              -$120
05/19  CONTRACT ADJ              -$40
05/19  PT RESP                   -$20
```

This immediately feels enterprise-grade.

---

# 3. Use expandable ledger trees

PracticeSuite/Epic style:

```text
▶ Claim 90837 DOS 05/01
```

Expand:

```text
Charge                $180
Insurance Payment    -$120
Adjustment            -$40
Patient Resp          -$20
Balance                 $0
```

This is CRITICAL psychologically.

It gives users:

* financial traceability
* confidence
* auditability

---

# 4. Add unapplied cash rail

Epic-style systems ALWAYS expose unapplied cash clearly.

Add persistent side rail:

```text
--------------------------------
ERA Total        $1,248.22
Applied          $1,206.04
Unapplied         $42.18
--------------------------------
```

This should stay visible while posting.

---

# 5. Inline posting workflow

DO NOT use modal-heavy workflows.

Epic systems post inline.

Meaning:

```text
[ Match Claim ]
[ Apply Adjustment ]
[ Transfer ]
```

directly in ledger rows.

This is MUCH faster.

---

# 6. Add keyboard-first workflows

This is one of the BIGGEST differences.

Enterprise billers hate mouse-heavy systems.

You need:

```text
Tab = next field
Enter = apply
Space = expand claim
Cmd+K = search claim
```

This instantly makes it feel premium.

---

# 7. Introduce balancing indicators

Epic-style posting constantly reassures users.

Examples:

```text
✓ ERA balanced
```

or

```text
$42.18 remaining to post
```

with live updating.

The user should ALWAYS know:

* what remains
* what posted
* whether ledger balances

---

# 8. Show payer adjustment logic

Enterprise systems expose:

* CARC codes
* RARC codes
* contractual adjustment logic

Example:

```text
CO-45 Contractual obligation
```

This makes the software feel real-world billing capable.

---

# 9. Add split-screen posting mode

THIS is huge.

---

## Left

ERA lines

## Right

Patient ledger / claim ledger

Then users can:

* drag/apply
* compare balances
* reconcile visually

That’s exactly how advanced PM systems operate.

---

# 10. Add posting intelligence

This is where you can surpass Epic.

Example:

```text
Suggested posting:
- Match to DOS 05/01

Confidence: 98%
```

This is the future.

Epic is still heavily manual.

You can leapfrog them.
