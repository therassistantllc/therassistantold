Section 1
Module Definitions

Each module is independently deployable with defined boundaries, data ownership, and role access. Modules communicate via internal service events and a shared data layer.

🏥
1 · Patient Portal
Patient-facing layer for intake, communication, payments, and self-service health management.
Core Features
Intake Forms Demographics Insurance Entry Appointment Requests Secure Messaging Telehealth Links Billing Statements Credit Card on File Document Upload PHQ-9 / GAD-7 PCL-5 / ASRS ACE / C-SSRS Consent Forms
Primary Users
Patient Front Desk
Data Owner
patient_profiles, intake_submissions, screening_scores, appointments_requests
Depends On
  • EHR Core — patient record linkage, appointment confirmation
  • Billing Engine — statement generation, payment capture via Stripe
  • External — Zoom (telehealth), Stripe (payments)
📋
2 · EHR Core
Central clinical record system. Source of truth for all patient encounters, treatment, and clinical documentation.
Core Features
Patient Records Scheduling Appointment Calendar Recurring Appointments Treatment Plans Progress Notes Intake Assessments Group Notes Family Session Notes Psychiatric Evaluations Telehealth Tracking Locked / Finalized Records Supervisor Signatures Addendum Workflows
Primary Users
Clinician Supervisor Front Desk Admin
Data Owner
encounters, progress_notes, treatment_plans, appointments, diagnoses
Depends On
  • Patient Portal — intake data, demographics, screening scores
  • Coding & Revenue — triggers coding engine on note save/finalize
  • External — Google Calendar (sync), Zoom (session links)
⚡
3 · Coding & Revenue Capture
Autonomous coding layer that analyzes clinical data and surfaces CPT/HCPCS/ICD-10 recommendations, missed codes, and documentation gaps.
Core Features
CPT Support HCPCS Support ICD-10 Support Smart Charge Suggestions Trigger Logic Engine Medical Necessity Prompts Revenue Capture Prompts Charge Review Queue Charge Capture Doc Support Prompts Missed Code Detection Audit-Ready Reports Colorado Medicaid Rules
Primary Users
Clinician Billing Specialist Admin
Data Owner
coding_reports, charge_items, code_trigger_rules, revenue_capture_events
Depends On
  • EHR Core — session data, note type, diagnostic codes, modifiers
  • Billing Engine — sends approved charges for claim creation
  • Credentialing — validates provider NPI / taxonomy before charge capture
💳
4 · Billing Engine
Full revenue cycle management from eligibility verification through payment posting, denials, patient balances, and write-offs.
Core Features
Eligibility Verification Deductible / Copay Check Prior Authorization Referral Tracking Claim Creation Claim Scrubbing Batch Submission Claim Status Monitoring ERA Posting EFT Tracking Denials & Appeals Patient Balances Payment Plans Auto-Pay Statements Refunds & Write-offs
Primary Users
Billing Specialist Admin
Data Owner
claims, era_imports, payments, patient_balances, prior_auths, eft_records
Depends On
  • Coding & Revenue — approved charge items to create claims
  • EHR Core — encounter dates, provider NPI, place of service
  • External — Availity, Office Ally, Stripe, Colorado Medicaid, payer EDI
🛡️
5 · Credentialing & Compliance
Tracks provider enrollment, license expiration, malpractice coverage, contract status, HIPAA compliance, and audit readiness.
Core Features
Provider Enrollment Credentialing Status Recredentialing Reminders CAQH Tracking License Expiration DEA Expiration Malpractice Tracking Contract Status HIPAA Compliance Audit Logs ROI Tracking Consent Tracking Medicaid Compliance CO Mental Health Law
Primary Users
Credentialing Spec. Admin Supervisor
Data Owner
providers, credentials, licenses, contracts, audit_logs, consent_records
Depends On
  • External — CAQH, PECOS, NPPES, state licensing boards
  • Billing Engine — validates active enrollment before claim submission
  • Reporting — feeds compliance dashboard and expiration alerts
📊
6 · Reporting & Analytics
Cross-module intelligence layer providing revenue dashboards, clinical outcomes, compliance metrics, and operational reporting.
Core Features
Revenue Dashboards Productivity Dashboards Claims Aging Auth Expiration Reports Revenue by Provider Revenue by Payer Outstanding Balances Patient Retention No-Show Tracking Screening Score Trends Audit Findings Compliance Reports Saved Reports
Primary Users
Admin Billing Specialist Supervisor Credentialing
Data Owner
report_definitions, saved_reports, analytics_snapshots (read from all modules)
Depends On
  • All modules — read-only aggregation; no write-back to source modules
  • Billing Engine — claims, ERA, payment, denial data
  • EHR Core — encounter volume, screening trends, no-shows
  • Credentialing — compliance status, expiration timelines
🔌
7 · External Integrations
Managed integration layer. All third-party connections are routed through adapter services with auth, retry, and audit logging. No module calls external systems directly.

Architecture Notes
  • All integrations use OAuth 2.0 or API key vault storage
  • Webhook receivers are isolated and rate-limited
  • EDI 270/271 (eligibility), 837 (claims), 835 (ERA) supported
  • Failure queues with admin alert and retry logic

Cross-Module Data Flows

Patient Portal EHR Core Billing Engine Demographics, insurance info, consent forms, intake screening scores, appointment requests Patient submits intake / books appointment

EHR Core Reporting Encounter volume, appointment outcomes, no-shows, screening score trends Nightly aggregation snapshot
Coding & Revenue Billing Engine Approved charge items: CPT, HCPCS, ICD-10, units, modifiers, revenue code Charge review completed / auto-approved
Coding & Revenue Reporting Coding audit events, missed code flags, revenue capture opportunities, audit report data Coding report generated
Credentialing Coding & Revenue Billing Engine Active provider status, NPI, taxonomy codes, payer enrollment status, contract effective dates Provider validation check at charge capture and claim creation

Credentialing Reporting License expiration timeline, enrollment gaps, HIPAA audit log entries, compliance flags Credential record updated / expiration alert fired
